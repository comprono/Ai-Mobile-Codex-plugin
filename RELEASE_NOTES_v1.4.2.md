# AI Mobile 1.4.2

AI Mobile 1.4.2 fixes the integration gaps that made the first real production Director-CFO activation stop before context work.

The coordinator now receives the same free-disk shape exposed by the public inventory, keeps Spark and general Codex quota pools separate, and recognizes `run-command` as the authorization for command-capable observation. Complete context, strategy, execution, and reconciliation contracts reach workers without router truncation. Long Antigravity prompts use an isolated temporary prompt file instead of Windows process arguments, avoiding command-length failure and process-list context exposure.

Master Plans must link every unresolved project requirement by exact ID. Worker-authored claims cannot pass those requirements without package-linked deterministic proof; exhausted plans create an owned verification transition and then strong reconciliation when needed.

Existing Director programs resume in place after an upgrade. The restart handoff verifies both semantic version and runtime fingerprint, requires exactly one campaign for the same durable task, and rejects reconciliation, migration, start, or legacy-cycle tools.

Read-only context and strategy workers run from immutable snapshots; SQLite WAL state is captured through an online backup, full-tree mutations fail closed, resource leases account for RAM and disk, and an all-failed Director round always enters strong reconciliation instead of retaining stale ownership.

The deterministic release matrix contains 35 fail-fast Node checks (34 regression or end-to-end scripts plus the self-test), 98 JavaScript syntax checks, and 55 MCP self-test assertions. The mutation-guarded clone of the exact Job Vibhu production state passed real context integration, frontier strategy, a three-milestone/seven-workstream Master Plan, whole-plan budgeting, and capture of the first execution-phase route without launching it or changing production.
