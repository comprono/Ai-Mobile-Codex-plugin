# AI Mobile v0.3.1 - root-outcome delivery batches

AI Mobile now keeps the complete project outcome active while Codex and bounded workers advance through dependency-ready milestones.

## Changes

- A passing milestone is now an integration point, not a stopping condition.
- Broad goals use a compact two-to-four item delivery batch tied to explicit completion evidence.
- Current Codex remains on the critical path and continues immediately after dispatch.
- Healthy independent provider capacity is used when the handoff saves time or context.
- Worker handoffs receive both the immutable project outcome and their narrow lane boundary.
- `Next` means work already started or the exact blocker that prevented it.
- No manager loop, heartbeat, recurring automation, repeated status polling, or automatic desktop launch is reintroduced.

## Verification

- Full bridge self-test.
- Portable MCP initialization from a copied path containing spaces.
- Public marketplace and strict security scanner profiles.
- Live development-task exercise against a measurable throughput objective.

Restart Codex after updating so existing tasks receive the new skill contract.
