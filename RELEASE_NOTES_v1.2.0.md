# AI Mobile 1.2.0 - lightweight Codex console, capacity-aware AI work plane

AI Mobile 1.2.0 corrects the core execution model.

The visible Codex task is now a lightweight project console. It receives direction, invokes the deterministic coordinator, presents decisions, and reports verified material progress. It owns no project files and does not perform bulk reading, architecture, coding, integration, or expensive review.

Actual project work is assigned to separate Codex CLI, Claude Code, Antigravity CLI, or optional Cursor workers from fresh task-fit, dependency, quota, reset-horizon, reliability, cost, RAM, storage, and user-priority evidence. Shared Codex capacity above the configured reserve can be used productively through Codex CLI.

Key changes:

- First-class console and work-plane runtime contracts.
- Coordinator-generated dependency-ready units.
- Dynamic role-based provider and model selection.
- Deterministic one-time isolated patch integration.
- Boundary and concurrent-primary-change protection.
- Required primary-workspace verification and rollback on failure.
- No Luna or lower-tier model re-review merely to apply a verified patch.
- Strict plugin upgrade order using the official local Codex app-server: validate and install, close only OpenAI.Codex, prove the fresh runtime on the capable model, then start the same task on the lightweight console and reopen it.
- No automatic provider UI, manager loop, Goal, heartbeat, schedule, hidden CLI continuation, or repeated poll.

Release acceptance includes complete local regression, MCP lifecycle, portfolio, lease, worktree, trusted-model, restart, and plugin-scanner gates.
