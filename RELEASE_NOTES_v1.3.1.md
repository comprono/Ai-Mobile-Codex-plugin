# AI Mobile 1.3.1 - durable event-driven AI resource orchestration

AI Mobile 1.3.1 turns the visible Codex task into a lightweight project console while a finite local coordinator continues real project work through separate authenticated providers.

- Starts or reuses one detached event-driven coordinator with no LLM manager loop, heartbeat, Goal, automation, repeated polling, hidden continuation, or automatic desktop launch.
- Persists revisioned monotonic job and coordinator states, reconciles a lost worker process without waiting for its deadline, and performs bounded Windows cleanup without hiding a retained worktree.
- Discovers callable Codex CLI, Claude, Antigravity, and optional Cursor capacity, including live model availability, capability tier, authentication, subscription/API mode, quota pools, reset horizon, reliability, RAM, storage, and the protected Codex reserve.
- Coordinates one project or a portfolio of separate projects with machine-wide provider, quota, worker, file, and storage leases.
- Converts compact read-only observations into exact dependency-ready writer units and advances them without another visible model turn.
- Keeps writers isolated in Git worktrees. Codex workers return bounded diffs that are recounted, path-checked, unsafe symlink/submodule modes refused, tested, and integrated without sandbox bypass.
- Reports only material assignments, accepted evidence, typed blockers, resource decisions, and owned next actions through passive material status.
- Adds privacy-safe provider diagnostics and exposes both new surfaces to Codex, Claude Code, and the PowerShell helper.
- Passes 21 local regression gates plus a real concurrent Codex/Claude two-project portfolio canary with independent acceptance, 15 percent Codex reserve protection, complete worktree cleanup, and no desktop launch.

Normal execution never restarts Codex. A restart is required only after installing this schema upgrade into an already open host task.
